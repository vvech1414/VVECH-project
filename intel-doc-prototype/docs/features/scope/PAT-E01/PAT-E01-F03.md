# PAT-E01-F03 — Basic profile setup

> Эпик: **PAT-E01** Партнёр-контекстный онбординг пациента
> Поверхность: Приложение пациента (PAT) · Путешествие: Онбординг пациента (J1)

## 1. Что делает фича

- **Шаг:** 3
- **Экран:** Basic profile setup
- **Действия пользователя:** Заполняет имя, дату рождения, контекст состояния здоровья; сохраняет
- **Данные системы:** name, dob, conditionContext, contact (mock state)
- **Happy path:** Заполняет минимальный профиль и переходит дальше
- **JTBD:** Передать минимальные данные о себе без перегруза формой

## 2. Контекст эпика

- **Initiative flow:** QR/ссылка от ЭНЦ → подтверждение партнёра → согласия → базовый профиль → выдача доступа ЛПУ → главный экран
- **JTBD functional:** Войти в приложение и убедиться, что нахожусь в правильном медицинском контексте партнёра
- **JTBD emotional:** Доверие к новому каналу взаимодействия с клиникой
- **Состояние эпика в прототипе:** Готово
- **Комментарий из реестра:** Реализован 4-экранный flow v4 (welcome → consent → profile → access)

## 3. Применимые гардрейлы

- **Прототип, не продакшн** — оптимизация под ясность демо, mock-state, без реальных API/OCR/auth/БД.
- **AI consultative only** — никаких diagnosis/treatment/prescription формулировок; разрешены summary, clarification, prep-guidance, neutral explanations.
- **Прозрачность доступа** — пациент видит recipient, scope, срок, статус доступа; никакой «тихой» передачи данных.
- **Партнёр-first UX** — ЭНЦ-контекст виден в экране, ощущение клинико-связанного канала.
- **A11y и возраст 35–65** — крупные tap-targets, читаемый шрифт, низкая когнитивная нагрузка.
- **Copy на русском** — короткие лейблы, supportive helper-тексты, без алармизма.
- **Reusable components** — использовать компоненты из `src/components/` (см. §6).

## 8. Copy guidance

**Тон:** спокойный, практичный, не алармистский. Короткие лейблы, поддерживающий helper-текст.

**Хорошие примеры:**
- «Сохранить профиль»
- «Это поможет врачу подготовиться к приёму»
- «Только базовые данные, можно дополнить позже»

**Избегать:**
- «диагноз»
- «назначение лечения»
- «рецепт» как ключевая функция AI/app
- алармистские формулировки
