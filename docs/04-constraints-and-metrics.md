## 1. Regulatory Constraints

### Российская юрисдикция

**152-ФЗ "О персональных данных"**
- Сбор и обработка ПДн только с явного письменного / электронного согласия
- Раздельные согласия на разные цели обработки (передача врачу ≠ маркетинг)
- Право пользователя на удаление, исправление, отзыв согласия
- Уведомление Роскомнадзора об обработке ПДн (если применимо к финальному продукту)
- Локализация серверной инфраструктуры на территории РФ (ФЗ-242)

**ФЗ-323 "Об основах охраны здоровья граждан"**
- Медицинские данные относятся к специальной категории ПДн
- Передача мед-данных между мед-учреждениями требует юридического основания
- Прототип IntelDoc — **не является медицинским изделием** и не предназначен для постановки диагнозов

**Прочее**
- Прототип на защите фигурирует как **educational prototype**, не commercial product
- Реальные медицинские данные пациентов не используются — только синтетические / анонимизированные test data

### Implications для дизайна и кода

- Любая UI-формулировка типа "приложение рекомендует..." должна быть переработана в "обратите внимание на..." — продукт не даёт медицинских рекомендаций
- Прайвеси-полиси и пользовательское соглашение обязательны (драфт уровня — TODO)
- В прототипе достаточно демонстрировать **архитектуру consent** и audit log, не нужны реальные юридические тексты

## 2. Technical Constraints

### Stack constraints
- **Frontend:** React (existing prototype) — нельзя переписать на другом фреймворке
- **Deployment:** Vercel (для внешней ссылки на защиту)
- **Mobile:** прототип — **mobile-first web (Vite + React + Tailwind)**, разрабатывается под 390 px iPhone, разворачивается как обычный SPA на Vercel. React Native не используется. Web-app оптимизирован для просмотра с телефона; PWA-манифест на текущем этапе не подключён.
- **Backend:** реальный backend отсутствует. Все данные — in-memory Zustand store (`src/store/`) + детерминированные mock-фикстуры (`src/data/mockData.ts`, `src/data/adminMockData.ts`). OCR имитируется через `src/lib/ocr-mock.ts`.

### Resource constraints
- **Team:** Один разработчик (автор защиты, соло-формат) + AI-инструменты
- **Budget:** только подписки на инструменты (Claude, Vercel free tier, Figma free)
- **Time:** **дата защиты — 16 мая 2026 (2026-05-16)**. До защиты — 3 дня от текущей точки, что делает roadmap по этапам жёстким: фокус на стабилизации Track A + сборку design-system артефактов + финализацию слайдов.

### Scope constraints
- **MVP-only:** не all-17-initiatives, focus на patient-side core (см. `docs/01-scope-and-vision.md`)
- **No real backend integration:** mock data, no real medical APIs
- **No real clinic data:** synthetic test data only

## 5. Success Metrics

### Product Success Metrics (для гипотетического запуска)

**North Star Metric:**
- % приёмов в клинике, на которые пациент пришёл с данными из IntelDoc

**Leading indicators:**
- DAU / WAU / MAU
- Activation: % новых пользователей, загрузивших ≥ 1 документ за первую неделю
- Retention D7 / D30
- Avg documents per active user per month
- % пациентов, использовавших share-flow хотя бы раз

**Quality metrics:**
- OCR accuracy на golden dataset ≥ 90%
- Crash-free sessions ≥ 99.5%
- Time-to-first-document ≤ 5 минут от регистрации

**Trust / compliance metrics:**
- 0 нарушений 152-ФЗ
- 100% действий с данными в audit log
- % пользователей, отозвавших consent → должна быть низкая

### Defense Success Metrics (для образовательной защиты)

**Артефакты сданы:**
- [ ] Кликабельный прототип доступен по public URL
- [ ] Презентация загружена / показана
- [ ] Figma-файл доступен по ссылке

**Качественные критерии:**
- [ ] Каждый дизайн-decision объясняется через JTBD
- [ ] Видна end-to-end логика: интервью → требования → дизайн → код
- [ ] Design system явно отделена от screens (декомпозиция показана)
- [ ] Готовы ответы на 5 ожидаемых вопросов комиссии (см. CLAUDE.md Section 10)

**Дифференциация:**
- [ ] Использованы AI-инструменты как часть нарратива (Claude Design, Figma MCP) — это сам по себе сигнал современного workflow
- [ ] Audit trail и consent-first архитектура показаны как ключевые дифференциаторы
